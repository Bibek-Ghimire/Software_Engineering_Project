AUTHENTICATION & USER MANAGEMENT
================================

1. User Registration
   Allows new users to create accounts with name, email, password, role selection
   (Student/Teacher), and learning interest selection. Passwords are hashed using bcrypt.

2. User Login
   Authenticates users with email/password and generates JWT tokens for session
   management with 1-hour expiration.

3. User Logout
   Securely logs out authenticated users and invalidates session tokens.

4. User Profile Management
   Students and teachers can view and update their profiles including profile
   pictures, bios, qualifications, skills, and interests with file upload support.

5. Role-Based Access Control
   Differentiates functionality between Student and Teacher roles with protected
   routes and specific permissions for each role type.

6. Student Profile
   Includes name, email, interests, skills, college, and batch assignment tracking.

7. Teacher Profile
   Includes department, qualification, subject, bio, achievements, resume, and
   social media links (GitHub, LinkedIn).


COURSE MANAGEMENT
=================

8. Course Creation & Management
   Teachers can create, edit, and manage courses with title, description, level
   (Beginner/Intermediate/Expert), duration, and price information.

9. Course Browsing
   Students can view all available courses with filtering by level, duration, and
   other criteria with teacher information displayed.

10. Course Details View
    Displays comprehensive course information including teacher profile, enrollment
    count, rating, and student enrollment status.

11. Course Enrollment
    Students can request to enroll in courses, triggering an enrollment request
    workflow with teacher approval required.

12. Enrollment Request Management
    Teachers receive and manage enrollment requests with ability to approve or
    reject with optional rejection reasons.

13. Payment Integration
    Processes course payments with status tracking (pending/completed/failed),
    transaction IDs, and failure reason logging.

14. Course Keywords & Recommendations
    Courses tagged with keywords for intelligent recommendation algorithm matching
    user interests.


STUDY BATCH ALLOCATION
======================

15. Intelligent Batch Allocation
    Automatically groups students into study batches based on shared interests and
    skills using multiple clustering algorithms (K-means, hierarchical clustering,
    greedy algorithm, DBSCAN, spectral clustering).

16. Jaccard Similarity Calculation
    Computes similarity between users based on interests and skills using set
    intersection/union for batch matching algorithm.

17. Weighted Similarity Scoring
    Combines interest similarity (70% weight) and skills similarity (30% weight)
    to determine optimal batch groupings.

18. Batch Management
    Supports batch creation, member management, allocation/reallocation of users,
    and batch deletion with max size constraints.

19. User Batch Assignment
    Automatically assigns new students to appropriate batches upon registration
    and allows manual reallocation by administrators.

20. Batch Analytics
    Provides batch statistics including member counts, dominant interests per batch,
    and allocation overview data.

21. Batch Discovery
    Students can view their assigned batch, batch members, and similar users within
    their batch for collaboration opportunities.

22. Study Batch Page
    Dedicated interface showing user's current batch, batch members, and similar
    users for easy connection and collaboration.


RECOMMENDATION ENGINE
====================

23. Intelligent Course Recommendations
    Recommends courses based on user interests using Jaccard similarity with
    minimum score threshold (0.3) filtering and trending fallback.

24. Resource Recommendations
    Suggests study resources based on user interests and keywords matching.

25. Group Recommendations
    Recommends study groups based on shared interests and course details.

26. Teacher Recommendations
    Suggests relevant teachers based on their course specializations and user
    interests.

27. Interest-Based Content Matching
    Matches user interests with course titles, descriptions, and keywords using
    substring and word matching algorithms.

28. Trending Fallback System
    Returns trending/popular items when no personalized recommendations match user
    interests to ensure relevant content.

29. Recommendation Scoring
    Displays recommendation scores indicating relevance percentage for each suggested
    item to user interests.

30. Interested Courses Tracking
    Tracks which courses users are interested in for personalized recommendations
    and engagement metrics.
